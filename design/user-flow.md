## 1. 認證流程

### 1.1 Google 登入

```
[登入頁]
    │
    ▼
使用者點擊「使用 Google 帳號登入」
    │
    ▼
系統跳出 Google OAuth 授權視窗
    │
    ├─ 授權成功
    │       │
    │       ├─ 首次使用 ──────────────► [資料填寫頁 02] → 填寫暱稱 + 大頭貼 → [首頁 03]
    │       │
    │       └─ 已有帳號 ─────────────► [首頁 03]
    │
    └─ 授權取消 / 失敗
            │
            ▼
        留在登入頁，顯示錯誤提示
```

---

### 1.2 匿名登入

```
[登入頁]
    │
    ▼
使用者點擊「免登入開始」
    │
    ▼
頁面出現 Cloudflare Turnstile 驗證元件
    │
    ├─ 驗證通過
    │       │
    │       ▼
    │   系統建立 Firebase 匿名帳號
    │       │
    │       ├─ 首次使用 ──────────────► [資料填寫頁 02] → 填寫暱稱 → [首頁 03]
    │       │
    │       └─ 已有匿名帳號 ──────────► [首頁 03]
    │
    └─ 驗證失敗
            │
            ▼
        顯示失敗提示，可重新驗證
```

---

### 1.3 首次使用資料填寫

```
[資料填寫頁 02]
    │
    ▼
（選填）點擊大頭貼圓圈
    │
    ▼
底部彈出選項：「拍攝照片」/ 「從相簿選取」/ 「取消」
    │
    ├─ 選取圖片
    │       │
    │       ▼
    │   上傳中（Loading 轉圈）
    │       │
    │       ├─ 上傳成功 → 圓圈顯示預覽圖
    │       └─ 上傳失敗 → 顯示錯誤提示，保留預設頭像
    │
    └─ 取消 → 維持預設頭像
    │
    ▼
（必填）輸入暱稱
    │
    ▼
點擊「完成，開始使用！」
    │
    ├─ 暱稱為空 → 按鈕維持 disabled，不可點擊
    │
    └─ 暱稱有值
            │
            ▼
        資料寫入 Firestore → [首頁 03]
```

---

### 1.4 匿名帳號升級（綁定 Google）

```
[設定頁 11] → 點擊「綁定 Google 帳號」
    │
    ▼
系統跳出 Google OAuth 授權視窗
    │
    ├─ 授權成功
    │       │
    │       ▼
    │   匿名帳號與 Google 帳號合併
    │   原有資料（群組、分帳紀錄）完整保留
    │       │
    │       ▼
    │   設定頁隱藏「綁定 Google 帳號」選項
    │   顯示成功提示：「帳號已成功綁定」
    │
    └─ 授權取消 / 失敗
            │
            ▼
        留在設定頁，顯示失敗提示
```

## 2. 群組流程

### 2.1 建立群組

```
[首頁 03] 或 [群組列表頁 04]
    │
    ▼
點擊「＋ 新增」 或 底部 Tab「＋」→ 選「新增群組」
    │
    ▼
[建立群組頁 05]
    │
    ├─ （選填）點擊封面圖區域上傳封面
    │       │
    │       └─ 選取圖片 → 上傳至 R2 → 預覽顯示
    │
    ├─ （必填）輸入群組名稱
    │
    ├─ （選填）搜尋並新增成員
    │       │
    │       ├─ 從歷史聯絡人選取（按頻率排序）→ 加入成員列表
    │       │
    │       └─ 輸入新名字 → 點擊「+ 新增 XXX 作為成員」→ 加入未綁定佔位成員
    │
    ▼
點擊「建立群組」
    │
    ├─ 群組名稱為空 → 按鈕 disabled，不可點擊
    │
    └─ 驗證通過
            │
            ▼
        寫入 Firestore
        後台自動產生邀請連結（inviteCode）
            │
            ▼
        [群組詳情頁 06 - 總覽 Tab]
        顯示成功提示：「群組已建立，快邀請朋友加入！」
            │
            ▼
        （可選）分享邀請連結
```

---

### 2.2 透過邀請連結加入群組

#### 情境 A：未登入使用者點擊邀請連結

```
點擊邀請連結 https://zplit.app/join/xxxxxxxx
    │
    ▼
[加入群組頁 10 - 流程一]
顯示群組名稱、封面、成員數
    │
    ▼
使用者選擇登入方式（Google / 匿名）
    │
    ▼
完成登入 → 進入流程 B（選擇身份）
```

#### 情境 B：已登入使用者（含剛完成登入）

```
[加入群組頁 10 - 流程二]
顯示群組現有成員列表
    │
    ├─ 選擇現有佔位成員（未綁定）
    │       │
    │       ▼
    │   [流程三] 確認 / 修改顯示名稱
    │       │
    │       ▼
    │   系統將帳號綁定至該佔位成員
    │       │
    │       ▼
    │   [群組詳情頁 06]
    │
    └─ 點擊「以新成員加入」
            │
            ▼
        [流程三] 輸入顯示名稱
            │
            ▼
        系統新建一個成員身份
            │
            ▼
        [群組詳情頁 06]
```

#### 情境 C：邀請連結無效

```
點擊失效的邀請連結
    │
    ▼
顯示錯誤頁：「此邀請連結已失效或群組不存在」
提供按鈕：「返回首頁」
```

---

### 2.3 新增帳務

```
[群組詳情頁 06] 或 [個人分帳詳情頁 09]
    │
    ▼
點擊右下角「＋ 新增帳務」浮動按鈕
    │
    ▼
[新增帳務頁 07]
    │
    ├─ 輸入品項名稱（必填）
    ├─ 輸入金額（必填）
    ├─ 選擇付款人（預設為自己）
    ├─ 修改日期時間（預設為現在）
    │
    ├─ 選擇分帳模式（預設 Simple Split）
    │       │
    │       ├─ Simple Split：勾選分帳人 → 系統自動平均分配
    │       │
    │       ├─ Split by Amount：逐一輸入每人金額
    │       │       │
    │       │       └─ 即時驗證：總和 ≠ 帳單金額 → 顯示紅色警告
    │       │
    │       └─ Split by Percent：逐一輸入百分比
    │               │
    │               └─ 即時驗證：總和 ≠ 100% → 顯示紅色警告
    │
    ├─ （選填）展開「詳細資料」
    │       ├─ 輸入備注說明
    │       └─ 上傳收據圖片 → 上傳至 Cloudflare R2
    │
    ├─ （選填）展開「重複記帳」
    │       ├─ 選擇頻率（每日 / 每週 / 每月 / 自訂）
    │       └─ 設定結束日期
    │
    ▼
點擊「儲存」
    │
    ├─ 必填欄位未填 → 顯示欄位錯誤提示
    ├─ 分帳金額/百分比未對齊 → 顯示警告，無法儲存
    │
    └─ 驗證通過
            │
            ▼
        寫入 Firestore，觸發即時同步
        寫入 editLog 操作記錄
            │
            ▼
        返回 [群組詳情頁 06] 或 [個人分帳詳情頁 09]
        顯示成功提示
```

---

### 2.4 編輯 / 刪除帳務

```
[群組詳情頁 06 - 總覽 Tab]
    │
    ▼
長按或右滑帳務卡片
    │
    ├─ 選擇「編輯」
    │       │
    │       ▼
    │   進入預填資料的 [新增帳務頁 07]
    │       │
    │       ▼
    │   修改後點擊「儲存」
    │       │
    │       ▼
    │   更新 Firestore，寫入 editLog
    │   返回群組詳情頁，顯示成功提示
    │
    └─ 選擇「刪除」
            │
            ▼
        彈出確認對話框：「確定要刪除這筆帳務嗎？」
            │
            ├─ 確認刪除 → 從 Firestore 刪除，寫入 editLog → 列表移除該筆
            └─ 取消 → 關閉對話框
```

---

### 2.5 債務清算

```
[群組詳情頁 06] → 切換至「清算 Tab」
    │
    ▼
查看系統計算的最優轉帳方案列表

    ── 單筆清算 ──
    │
    ▼
點擊單筆「標記完成」
    │
    ▼
該筆顯示 ✅ 已完成，進度條更新
（可再次點擊取消完成狀態）

    ── 全部清算 ──
    │
    ▼
點擊「一鍵全部標記完成」
    │
    ▼
彈出確認提示：「確定要將所有帳款標記為已完成？」
    │
    ├─ 確認 → 所有項目標記完成，進度條達 100%
    │         群組帳務淨額歸零
    └─ 取消 → 不做任何變更
```

## 3. 個人分帳流程

### 3.1 新增個人分帳

```
[首頁 03] 或 [個人分帳總覽頁 08]
    │
    ▼
點擊「＋ 新增」
    │
    ▼
選擇對象
    │
    ├─ 從歷史聯絡人選取（按互動頻率排序）
    │
    └─ 輸入新朋友名字 → 點擊「+ 新增 XXX」→ 建立新聯絡人
    │
    ▼
進入 [新增帳務頁 07]（個人模式：付款人固定為自己）
    │
    ▼
填寫帳務資訊（同 2.3 流程）
    │
    ▼
儲存 → 寫入 Firestore
    │
    ▼
若對方已綁定 Zplit 帳號
    │
    └─ 觸發 Firestore onSnapshot 雙向同步
       對方 App 同步顯示反向帳務
    │
    ▼
返回 [個人分帳詳情頁 09]
```

---

### 3.2 結清帳款

#### 情境 A：個人分帳頁結清

```
[個人分帳詳情頁 09]
    │
    ▼
點擊右上角「⋯ 更多」→ 選擇「結清所有帳款」
    │
    ▼
彈出確認提示：「確定要將與 XXX 的帳款全數結清？」
    │
    ├─ 確認 → 系統新增一筆結清記錄，淨額歸零
    │         頁面顯示「✅ 已結清」狀態
    └─ 取消 → 不做任何變更
```

#### 情境 B：群組清算（見 2.5）

---

### 3.3 開啟最佳分帳（Optimal Split）

```
[個人分帳詳情頁 09]
    │
    ▼
點擊「最佳分帳」開關，切換為開啟
    │
    ▼
系統掃描雙方是否有共同參加過的群組
    │
    ├─ 找到共同群組
    │       │
    │       ▼
    │   頁面頂部顯示黃色提示：
    │   「找到共同群組：墾丁旅遊 2026」
    │   「已將個人分帳納入群組結算網格優化」
    │       │
    │       ▼
    │   清算方案重新計算，顯示優化後的轉帳建議
    │   （轉帳次數最小化）
    │
    └─ 未找到共同群組
            │
            ▼
        顯示提示：「目前未找到共同群組，維持一對一計算」
        開關維持開啟（日後若加入共同群組將自動套用）
```

## 4. 帳務詳情流程

```
[群組詳情頁 06 總覽 Tab] 或 [個人分帳詳情頁 09]
    │
    ▼
點擊帳務卡片
    │
    ▼
展開帳務詳情（Inline 展開 或 底部 Sheet）
    │
    顯示：
    ├─ 品項名稱
    ├─ 金額
    ├─ 付款人
    ├─ 各人分攤金額明細

    ├─ 備注說明（若有）

    │
    ├─ 點擊「編輯」→ 進入 [新增帳務頁 07]（預填資料）
    └─ 點擊「刪除」→ 確認刪除流程

## 5. 設定流程

### 5.1 修改個人資料

```
[設定頁 11] → 點擊「編輯」
    │
    ▼
[個人資料編輯頁（子頁面）]
    │
    ├─ 點擊大頭貼 → 選取圖片 → 上傳至 R2 → 預覽更新
    │
    ├─ 修改暱稱
    │
    ▼
點擊「儲存」
    │
    └─ 寫入 Firestore，所有群組內的顯示名稱同步更新
       顯示成功提示 → 返回設定頁
```

---

### 5.2 切換語言

```
[設定頁 11]
    │
    ▼
點擊語言下拉選單
    │
    ▼
選擇「繁體中文」或「English」
    │
    ▼
整個 App 介面語言即時切換（無需重新整理）
偏好儲存至 LocalStorage
```

---

### 5.3 切換主題

```
[設定頁 11]
    │
    ▼
點擊主題選項：「跟隨系統」/ 「淺色」/ 「深色」
    │
    ▼
App 介面即時套用對應 DaisyUI 主題
（淺色 = lemonade，深色 = dim）
偏好儲存至 LocalStorage，覆蓋系統預設
```

---

### 5.4 登出

```
[設定頁 11] → 點擊「登出」
    │
    ▼
彈出確認提示：「確定要登出嗎？」
    │
    ├─ 確認 → 清除 Firebase Session → [登入頁 01]
    └─ 取消 → 留在設定頁
```

---

### 5.5 刪除帳號

```
[設定頁 11] → 點擊「刪除帳號」（紅色）
    │
    ▼
彈出第一層確認：「這個動作無法復原，確定要刪除帳號嗎？」
    │
    ├─ 取消 → 關閉對話框
    │
    └─ 繼續
            │
            ▼
        第二層確認：輸入「刪除」文字確認
            │
            ├─ 文字不符 → 確認按鈕 disabled
            │
            └─ 文字正確 → 點擊確認
                    │
                    ▼
                刪除 Firestore 使用者資料
                刪除 Firebase Auth 帳號
                    │
                    ▼
                [登入頁 01]，顯示提示：「帳號已刪除」
```

### 6.1 重複記帳的後續管理

> 需求文件提到支援重複記帳，但未說明如何管理已設定的重複規則。

```
場景：使用者想取消或修改重複記帳設定

[群組詳情頁 06 總覽 Tab]
    │
    ▼
點擊帶有「🔁」圖示的重複帳務卡片
    │
    ▼
展開詳情，顯示「此為重複記帳，每週執行」
    │
    ▼
點擊「編輯」
    │
    ▼
進入 [新增帳務頁 07]，重複記帳區塊預填當前設定
    │
    ├─ 修改頻率或結束日期 → 儲存 → 套用至後續所有記錄
    │
    └─ 關閉重複記帳開關 → 儲存
            │
            ▼
        彈出提示：「要取消所有後續的重複記帳嗎？」
            ├─ 確認 → 後續記帳停止，已產生的記錄保留
            └─ 取消 → 不做變更
```

---

### 6.2 群組成員顯示名稱衝突

> 需求文件允許加入時修改群組內顯示名稱，但未處理名稱重複情境。

```
場景：群組內已有「小明」，新加入的人也想叫「小明」

[加入群組頁 10 - 流程三]
    │
    ▼
使用者輸入顯示名稱「小明」
    │
    ▼
系統偵測到群組內已有同名成員
    │
    ▼
顯示提示：「群組內已有成員叫做「小明」，建議使用其他名稱以避免混淆」
（非強制，使用者仍可選擇保留相同名稱）
    │
    ▼
使用者確認或修改後 → 完成加入
```

---

### 6.3 網路離線 / 同步失敗

> 需求文件強調即時同步，但未說明離線情境的處理方式。

```
場景：使用者在無網路環境下操作 App

[任意頁面]
    │
    ▼
偵測到網路中斷
    │
    ▼
頂部顯示黃色提示條：「目前離線中，操作將在恢復連線後同步」
    │
    ▼
